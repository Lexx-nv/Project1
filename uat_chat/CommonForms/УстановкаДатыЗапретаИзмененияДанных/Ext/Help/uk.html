Установка дати заборони зміни даних

 

У конфігурації підтримується можливість установки обмеження зміни даних раніше певної дати (далі "дата заборони").

Контроль обмеження зміни даних виконується для:

  • всіх документів при проведенні, перепроведенні, відміні проведення;
  • наборів записів регістрів накопичення, при цьому контроль виконується у всіх випадках зміни записів регістрів, включаючи програмне.

Для документів перевірочним значенням є дата цього документа, для наборів записів - період кожної із записів даного набору.

Після встановлення дати заборони зміни даних документи на вказану або більш ранню дату будуть доступні тільки для перегляду.

Обмеження по даті заборони не поширюється на користувачів, яким доступна роль "Повні права".

Можливі кілька варіантів установки дати заборони змін:

  • Загальна дата заборони змін. У цьому режимі встановлюється загальна дата заборони одночасно для всіх організацій і ролей користувачів.
  • Дата заборони по організаціях. Дозволяє встановити різні значення дат заборони для організацій, без поділу по ролях.
  • Дата заборони по організаціях і користувачам/групам користувачів. Дозволяє встановити різні значення дат заборони для організацій у розрізі користувачів і груп користувачів. Якщо із входить у кілька груп користувачів, як обмежуюча дата береться мінімальна з дат, установлених для груп користувача.

У випадку використання механізму дати заборони зміни даних, при спробі проведення документа (нового або існуючому непроведеного) у періоді до дати заборони, користувачеві виводиться повідомлення "У користувача недостатньо прав на виконання операцій над базою даних". Якщо ж виконується скасування проведення (перепроведення) існуючого документа до дати заборони, користувачеві виводиться повідомлення "Операція не виконана".
