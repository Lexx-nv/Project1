Обработка "АРМ механика"

Обработка "АРМ Механика" предназначена для автоматизации рабочего места механика. Основное назначение обработки -  анализ состояния транспортных средств (ТС), проведение ремонтов, работа с агрегатами ТС.  

Вся информация функционально сгруппирована по закладкам.

Закладка "Транспорт".

 Закладка предназначена для подбора ТС в "Заявку на ремонт". В списке "Транспортные средства" отображены ТС организации и их признаки – приближение срока тех-обслуживания (ТО) и просрочка документов.  С помощью кнопки  "Отбор ТС (Подошел срок ТО)"  можно включить отбор ТС по сроку приближающегося ТО, то есть в списке останутся только те Тс у которых подошел плановый срок ТО.  

 Двойной щелчок мыши на ТС в списке "Транспортные средства" осуществляет подбор его в поле "Автомобиль" на закладке "Ремонт".

В нижней части страницы отображены следующие информационные данные (сгруппированы по страницам):

·         Подошедшие ТО (отображает дополнительную информацию по ТО для выбранного ТС, например, вид ТО, параметры выработки, плановые сроки ТО и т.п.);

·         Установленные агрегаты (информация о установленных на ТС агрегатах – шинах и аккумуляторах);

·         Номенклатура ТС (информация об установленной на ТС в процессе ремонтов номенклатуре);

·         Проведенные ремонты (список ремонтных листов для  данного ТС).

 Закладка "Работы/Запчасти".

 Закладка предназначена для заполнения "Заявки на ремонт" данными по необходимым работам и запчастям. Кнопка "Вид подбора" позволяет переключать режим подбора (работы или запчасти). При этом для побора используется соответствующий справочник – "Работы по ремонту ТС" или "Номенклатура" соответственно.

Двойной щелчок мыши на элементе справочника заполняет одну из таблиц – "Работы по ремонту ТС" или "Запчасти для ТС". Заполняемая таблица определяется видом подбора.

При использовании подбора работ есть возможность включить отбор работ по ТС, если оно выбрано (в процессе работы с закладкой "Транспорт", либо вручную в поле "Автомобиль" на закладке "Ремонт".). Кнопка  "Включить отбор по" оставит в списке только те работы, которые определены для выбранного ТС.      

 Закладка "Ремонт".

 Закладка предназначена для формирования документа  "Заявка на ремонт" и или документа  "Ремонтный лист". Данные закладки могут быть заполнены как вручную, так и автоматически данными с закладок   "Транспорт" (подбор ТС) и "Работы/Запчасти" (списки необходимых работ и запчастей).

 Кнопка "Создать" формирует  документы, заполняя их данными данной страницы, при этом учитываются параметры группы "Создаваемые документы".  Таким образом, можно сформировать не только "Заявку на ремонт", а и "Ремонтный лист", указать необходимость печати создаваемых документов и количество экземпляров при печати, открыть формы создаваемых документов и очистить поля закладки "Ремонт". Кнопка "Очистить" очищает поля закладки "Задание".

 Закладка "Агрегаты".

Закладка предназначена для работы агрегатами ТС (шины). Левая верхняя таблица содержит информацию о моделях шин. Ниже выводятся остатки по выбранной модели шин с указанием места хранения, общих характеристик и серийного номера шины. Закладка "Фильтры" позволяет установить фильтрацию списка моделей по различным характеристикам, таким как, модель, сезонность и т.п.

Двойной щелчок мыши по модели шин позволяет указать информацию необходимую для поступления шин. Часть данных заполняется автоматически на основании информации о модели, а такие данные как начальный серийный номер и количество в серии (если серийные номера идут подряд, то можно ввести первый номер и количество шин) необходимо заполнить. Эти данные автоматически попадут в таблицу   "Шины" закладки "Поступление агрегатов".

Для упрощения работы с установкой/снятием шин разработана возможность установки/снятия шины используя механизм drag&drop("перетащил и бросил")  и графический макет автомобиля. При выборе ТС автоматически загружается его графический макет (заполняется в справочнике "Модели ТС"). Установить шину на макет можно перетащив мышью любую (необходимую) шину из таблицы остатков шин и бросив ее на нужное место шины на макете. Снять шину можно перетащив картинку шины с макета на таблицу "Установка/снятие агрегатов". Для перестановки шин местами необходимо перетащить картинки шин на вспомогательную таблицу справа от макета, а затем переносить шины из этой таблицы на новые места на макете.

Кнопка "Создать" формирует  документ "Ремонтный лист" автоматически заполняя все данные по проведенным операциям с шинами. Кнопка "Очистить" восстанавливает макет в актуальное состояние (загружает заново данные по установленным шина и очищает вспомогательные таблицы).

Для синхронизации данных с типовым регистром  "Контактная информация" предусмотрена кнопка "Обновить данными из контактов", расположенной на панели инструментов, либо в контекстном меню. Если в дереве "Контрагенты" выбран контрагент, то обновятся его данные, если группа, то все контрагенты этой группы. При обновлении нужно указать те виды контактной информации, которые подлежат синхронизации и/или обновлению.

 Закладка "Поступление агрегатов".

Закладка предназначена оформления документа "Поступление агрегатов". Заполняется вручную. Таблица "Шины" может быть заполнена автоматически, используя подбор на странице "Агрегаты".

Кнопка "Создать" формирует  документ "Поступление агрегатов " автоматически заполнив данными страницы. Кнопка "Очистить" очищает данные на странице.  

 Закладка "Планирование ремонтов".

Предназначена для создания заявок на ремонт и планирования ремонтов ТС. Дата планирования и Дней до ТО составляют интервал, в котором производится поиск подошедших и просроченных ТО. Поле Организация - отбор по организации, которой принадлежит ТС. Остальные поля представляют собой реквизиты, заполняемые при создании документов.

Таблица "Приближение ТО" содержит список ТС, по которым в ближайшем времени потребуется ТО или ТО просрочено. Заполняется таблица по кнопке "Заполнить". По кнопке "Создать документы" будут созданы отмеченные галочками в таблице документы. При создании с документами можно произвести стандартные для АРМ действия, указанные в рамке "Создаваемые документы". В колонке "Параметры ТО" коротко строкой перечислены основные показатели ТО. Нажав на гиперссылку можно перейти к форме настройки Прохождения ТО в форме модели ТС. Некоторые поля в таблице доступны для редактирования и представляют собой реквизиты, заполняемые при создании документов.

Диаграмма "Состояния ТС" отображает все текущие состояния для ТС из таблицы "Приближение ТО". По диаграмме может быть указан правильный интервал ремонта, который не будет пересекаться с другими документами, изменяющими состояние ТС (путевой лист, разнарядка и т.д.).

Цвета раскраски интервалов диаграммы Ганта:

  •           - в рейсе, сформирована разнарядка;
  •           - в ремонте;
  •           - простои или ТС выбыло;
  •           - запланирован;
  •           - на консервации;
  •           - прочее состояние;
  •           - пересечение интервалов состояний ТС.
