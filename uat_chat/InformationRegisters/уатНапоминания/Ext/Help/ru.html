Регистр сведений "Напоминания"

Назначение

Регистр сведений "Напоминания" предназначен для хранения существующих напоминаний.

Напоминания предназначены для оповещения о повторяющихся событиях, таких как дни рождения, праздники, памятные даты и т.п. Для каждого события может быть установлена дата и период повторения. Кроме того, Вы можете установить интервал заблаговременного напоминания о наступлении события.

Параметры

Периодичность:Непериодический

Режим записи: Независимый

Измерения

  • Пользователь. Ссылается на справочник «Пользователи». Реквизит предназначен для хранения ссылки на получателя напоминания.
  • Автор. Ссылается на справочник «Пользователи». Реквизит предназначен для хранения ссылки на автора напоминания - (текущего пользователя), который создал напоминание.
  • Завершено. Тип значения: «Булево». Реквизит предназначен для хранения флага завершенности напоминания.
  • Реальная дата оповещения. Тип значения: «Дата». Реквизит предназначен для хранения реальной даты когда будет выведено напоминание.
  • Объект. Тип значения: Любая ссылка. Ссылка на объект конфигурации, прилагаемый к напоминанию.

Ресурсы

  • Дата актуальности. Тип значения: «Дата». Дата до которой напоминание считается актуальным. После этой даты даже не завершенные напоминания выводится не будут.
  • Дата начала. Тип значения: «Дата». Дата оповещения как она заложена в напоминании без учета отсрочки напоминания пользователем и предварительного вывода напоминания.
  • Дата оповещения. Тип значения: «Дата». Дата очередного напоминания, для непериодических напоминаний равняется дате начала как она заложена в напоминании без учета отсрочки напоминания пользователем и предварительного вывода напоминания.
  • Описание. Тип значения: «Строка». Описание напоминания - сообщение содержащее суть напоминания.
  • Редактирование. Тип значения: «Булево». Индикатор редактирования объекта. Предназначен для запрета открытия и вывода напоминания во время редактирования.
  • Срок до начала. Тип значения: «Число». Предназначен для хранения срока в секундах за который нужно предупредить о наступлении события.
  • Тема. Тип значения: «Строка». Заголовок окна, который будет выведен при отображении напоминания.
  • Тип периода. Тип значения: «Строка». Строка хранящая тип периода напоминания (1 -8) символ – тип периодичности дня, (9 - 14) cимвол тип периодичности недели, (15 - 16) cимвол тип периодичности месяца, (17 - 18) cимвол тип периодичности года .
  • Удалить по истечении срока. Тип значения: «Булево». Признак удаления напоминания после истечения даты актуальности, или после завершения(для не периодических напоминаний).

Дополнительная информация

Периодичность опроса напоминаний устанавливается в константе "Интервал проверки напоминаний". Значение этой константы - количество минут, через которое производится поиск актуальных напоминаний. Если значение этой константы равно нулю, то поиск напоминаний не производится.
