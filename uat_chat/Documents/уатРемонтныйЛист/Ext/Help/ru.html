Документ "Ремонтный лист"

Назначение

Документ «Ремонтный лист» служит для учета выполненных ремонтов транспортных средств, отражения факта установки/снятия шин, аккумуляторов и прочих агрегатов, а так же является основанием для списания запасных частей и материалов.

Движения по регистрам выполняют документы с состояниями: "Исполнен" и "Закрыт", при этом движения выполняются на дату окончания ремонта.

Если для данного документа были сформированы соответствующие документы типовых конфигураций 1С, то такой документ уже нельзя изменить. Для возможности внесения изменений необходимо удалить сформированные типовые документы. Это может сделать только пользователь с правами «Администратор управления автотранспортом» из меню «Сервис»-«Регламентные операции УАТ»-«Список сформированных документов типовой конфигурации для УАТ». Необходимо найти нужный документ УАТ и нажать клавишу Del. Все сформированы для него документы типовой конфигурации будут помечены на удаление.

Реквизиты

  • Транспортное средство. Ссылается на справочник «Транспортные средства». Неисправное ТС, подлежащее ремонту.
  • Способ ремонта. Тип значения: Перечисление «Способы ремонта». Собственная ремонтная зона или сторонний автосервис; 
  • Склад / Автосервис. Ссылается на справочник «Склады», справочник «Контрагенты». Значение зависит от значения реквизита «Способ ремонта». Если способ ремонта  - «Собственная рем. зона», то склад, с которого будет производится отпуск запасных частей. Если способ ремонта - «Автосервис», то поставщик запчастей - контрагент.
  • Вид ремонта. Ссылается на справочник «Виды обслуживания ТС». Вид обслуживания / ремонта ТС.
  • Дата начала, дата окончания. Тип значения: «Дата». Планируемые дата и время начала и окончания ремонта. Ремонт изменяет состояние ТС в значение "В ремонте".
  • Заявка на ремонт. Ссылается на документ «Заявка на ремонт». Заявка на ремонт, по которой выполняется ремонтный лист;
  • Причина обращения. Тип значения: «Строка». Описание причины обращения.
  • Выполнен. Тип значения: «Булево». Флаг выполнения ремонтного листа. Если документ не выполнен, то движения по регистрам не производятся, кроме регистра Состояние ТС.

Закладка «Работы»

Содержит описание выполненных работ и количество затраченных часов. Если выбран способ ремонта «Автосервис», то появляются дополнительные колонки для указания стоимости работ. 

Закладка «З/ч и материалы»

Аналогична одноименной закладке заявки на ремонт, но содержит также добавочные стоимостные реквизиты. Если выбран способ ремонта «Автосервис», то появляются дополнительные колонки для указания стоимости работ. 

Закладки «Шины», «Аккумуляторы» и «Прочие агрегаты»

Содержат данные по агрегатам, а также информацию о количестве замененных агрегатов, а также о выполненной операции (реквизит «Состояние»). На этих закладках реквизит «Количество» недоступен для изменения, и всегда равен 1.

Закладка «Исполнители»

Содержит сведения об исполнителях выполненных работ и таблицу начисленной зарплаты. Автоматический расчет зарплаты производится при нажатии кнопки "Заполнить" над табличной частью "Зарплата". Расчет выполняется на основании отработанного времени из таблицы "Время работы". Перед расчетом зарплаты для водителя должен быть задан тариф с выбранным видом начисления "За участие в ремонте".

Закладка «Измененные показания приборов»

Вносится информация по измененным показателям спидометра и счетчика моточасов.

Способ вызова

Вызывается из меню "Ремонты --> Ремонтные листы"

Движения

  • «Агрегаты ТС». Выполняет приход или расход агрегатов на ТС, в зависимости от вида выполненной операции с агрегатом.
  • «Остатки агрегатов». Списывает агрегаты со склада агрегатов.
  • «Номенклатура ТС». Выполняет приход материалов на ТС - это могут быть расходные материалы, замененные элементы конструкции ТС и т.п.
  • «Партии товаров на складах». Списывает материалы со склада материалов.
  • «Показания спидометров и счетчиков ТС». Изменяет показания счетчиков и спидометров ТС и оборудования.
  • «Прохождение ТО». Вносит данные в регистр сведений о времени прохождения планового ТО.
  • «Состояние ТС». Добавляет запись в регистр сведений о том, что ТС находится в ремонте.
  • «Рабочее время сотрудников». Добавляет записи о наработке сотрудников для последующего отражения в табеле учета рабочего времени.
  • «Основные начисления». Выполняет расчет начислений зарплаты сотрудникам, добавляет записи по регистру расчета.
  • «Заявки на ремонт». В случае, если указана завка на ремонт, то выполняет расход работ по указанному ТС.
